# Changelog

Все значимые изменения в проекте `webtlo` документируются в этом файле.

Формат основан на [Keep a Changelog](https://keepachangelog.com/ru/1.1.0/),
и проект следует [Semantic Versioning](https://semver.org/spec/v2.0.0.html).

### [Список релизов](https://github.com/keepers-team/webtlo/releases)

---

### [3.8.1] - 2026-04-08
- Исправлено получение раздач от клиентов, которые присылают не валидные символы.

### [3.8.0] - 2026-04-01
- Добавлено использование `get-limit` API форума (было потеряно в версиях 2.5 - 2.6).
- Прочие исправления связанные с API форума.
---

### [3.7.6] - 2026-03-25
### [3.7.5] - 2026-03-18
### [3.7.4] - 2026-03-04
### [3.7.3] - 2026-02-25
- Борьба с последствиями изменений чтения конфигурации (emoji и параметры регулировки).

### [3.7.2] - 2026-02-19
- Добавлена валидация YAML файлов

### [3.7.1] - 2026-02-09
- Доработан алгоритм сортировки списка подразделов для двух языков.

### [3.7.0] - 2026-01-27
- Реструктурировано хранение и получение значений конфигурации приложения.
- Оптимизировано ведение журнала.
---

### [3.6.0] - 2025-10-10
- Внесены изменения в хранение спецсимволов в конфигурации.
---

### [3.5.6] - 2025-09-16
### [3.5.5] - 2025-09-10
- Убран неактуальный адрес форума.

### [3.5.4] - 2025-06-18
- Исправление цвета "пульки" для новых раздач при импорте истории средних сидов.

### [3.5.3] - 2025-06-12
### [3.5.2] - 2025-05-29
- Регулировка раздач с использованием статичного файла из API отчётов.

### [3.5.1] - 2025-05-25
- Исправлен алгоритм определения смены даты, для обновления истории средних сидов.

### [3.5.0] - 2025-05-23
- Использование API отчётов для загрузки сведений о подразделах и отчётах.
- Использование статичных файлов, где это возможно.
- Добавлена проверка прав доступа к API.
- Добавлена загрузка истории средних сидов.
- Добавлен ограничитель количества запросов в секунду к API форума.
- Версия БД 13 => 15.

### [3.4.0] - 2025-05-14
- Обновлены ссылки на отчёты из сводного отчёта.

### [3.3.2] - 2025-04-04

### [3.2.3] - 2024-10-22
- Исправление удаления файлов при удалении раздачи
- Добавлен фильтр раздач "с ошибкой в клиенте"
- Исправление запуска задач по расписанию для standalone.
- Добавлена опция для исключения "своих" раздач из отправляемых отчётов.
- Исправление отправки сводного отчёта, в случае если найдено более одного сообщения в теме.
- Убрана обязательность логина-пароля для Qbittorent, добавлено уведомление об этом, если не удалось авторизоваться.
- Добавлена проверка наличия пароля для торрент-клиентов, у которых они обязательны (qbit, Deluge, Flood).


### [3.2.2] - 2024-10-18
- Исправлена работа опций "Автоматизация и дополнительные настройки > Задачи, запускаемые из планировщика"

### [3.2.1] - 2024-10-05
- Исправлено сломанное webApi qBit 5.0+

### [3.2.0] - 2024-09-17
- Изменение алгоритма регулировки. Теперь используется API отчётов.
- Добавлена опция "динамической регулировки".
- Добавлен принудительный запуск давно не сидируемых раздач.

### [3.1.1] - 2024-07-29
### [3.1.0] - 2024-07-26
- Переписан функционал работы с форумом и авторизация.
- Переписан функционал скачивания и редактирования торрент файлов.
- В отправляемые отчёты добавлена дата отчёта и исправлена ошибка определения даты в сводном отчёте.
- Добавлена опция "Отправка отчётов" > "Снимать отметку хранения с не хранимых раздач".

### [3.0.0] - 2024-07-04
- Отключение возможности отправки отчётов на форум.
- Внедрение API отчётов (см 2.6.0).
- Удалены устаревшие модули.
---

### [2.6.2] - 2024-06-12
- Исправлена невозможность авторизации в Utorrent 3.*

### [2.6.1] - 2024-06-07
- Исправлена невозможность авторизации в Transmission 2.94.
- Исправлена проблема невозможности редактирования настроек клиента, при настройке с нуля.
- Добавлено использование порта по умолчанию (80/443), при авторизации в клиенте, если порт не указан в настройках клиента.
- Добавлена опция в настройки "Отправка отчётов" > "Снимать отметку хранения с не хранимых подразделов"

### [2.6.0] - 2024-06-02
- Внедрение API отчётов.
